Configuration

Some things in Checker can be configured in a checker.yml or checker.yml.dist file in the root of your project (the directory where you run the checker command). You can specify a custom config filename and location in composer.json or in the --config option of the console commands.

# cheker.yml
parameters:
  bin-dir: './vendor/bin'
  git-dir: '.'
  hooks-dir: ~
  hooks-preset: 'local'
  process-timeout: 60
  process-async-wait: 1000
  process-async-limit: 10
  stop-on-failure: false
  ignore-unstaged-changes: false
  strict: false
  progress: 'list'
  skip-success-output: false
  message:
    successfully: 'successfully.txt'
    failed: 'failed.txt'
  extensions: []
  tasks: ~
  git-hooks:
    commit-msg: ~
    pre-commit: ~
    pre-push: ~
  commands: ~

Use o sinalizador --no-verify( -n) quando você confirmar que ignora os ganchos pré-commit e commit-msg.

Ao executar ./vendor/bin/grumphp runtodos os arquivos no repositório são verificados e durante o gancho pré-commit depende da tarefa. A maioria das tarefas (afaik) só verifica arquivos phpcpdalterados, mas algumas tarefas ( por exemplo) irão analisar todo o projeto, uma vez que não seria útil se não.
